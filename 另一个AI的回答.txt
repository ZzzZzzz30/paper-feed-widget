```ts
setQueue: (queue) => set({ queue, currentIndex: 0 }),
```

```txt

再设置 currentIndex = saved.index
```

理论上这两步都执行完应该能恢复成功，但它不是一个原子更新，中间可能被其他渲染、effect、组件内部状态抢先消费。你现在看到“打开后第一篇不是上次文章，翻到下一篇才是上次文章”，就很像恢复时最终显示仍停在 `currentIndex = 0`，而不是 `saved.index`。`WidgetWindow` 的显示确实是通过 `queue[currentIndex]` 取当前文章的，所以只要 `currentIndex` 没恢复对，就会显示错。 store 中 `setQueue` 确实会重置索引。

## 一、在 store 里增加一个“原子恢复队列”的方法

修改 `src/store/index.ts`。

在 interface 里加：

```ts
restoreQueue: (queue: PushQueueItem[], index: number) => void
```

然后在 store 实现里加：

```ts
restoreQueue: (queue, index) =>
  set({
    queue,
    currentIndex: Math.min(Math.max(0, index), Math.max(0, queue.length - 1)),
  }),
```

也就是把这段：

```ts
setQueue: (queue) => set({ queue, currentIndex: 0 }),
```

保留给“新推送 / 刷新队列”用，但恢复历史状态时不要再用 `setQueue()`。

建议改成：

```ts
setQueue: (queue) => set({ queue, currentIndex: 0 }),

restoreQueue: (queue, index) =>
  set({
    queue,
    currentIndex: Math.min(Math.max(0, index), Math.max(0, queue.length - 1)),
  }),
```

## 三、不要只在退出时保存，应该在 `currentIndex` 每次变化时保存

你现在的保存逻辑只绑定了 `beforeunload` 和组件卸载：

```ts
window.addEventListener('beforeunload', save)
return () => { save(); window.removeEventListener('beforeunload', save) }
```

但 Electron 里你的关闭行为有 `mainWindow.hide()`，隐藏窗口不一定触发 `beforeunload`。而且 `savePushState` 是 `ipcRenderer.invoke()` 异步调用，程序退出时也可能来不及完成。你的 IPC 保存接口本身没有明显问题，确实把 `push_saved_ids` 和 `push_saved_index` 写入了 settings，并调用了 `saveNow()`。

建议把 `WidgetWindow.tsx` 里的“退出时保存”改成“索引或队列变化时保存”：

```ts
useEffect(() => {
  if (mode !== 'push') return
  if (queue.length === 0) return

  const ids = queue.map(it => it.articleId)
  window.electronAPI.savePushState({
    ids,
    index: currentIndex,
  })
}, [mode, queue, currentIndex])
```

原来的 `beforeunload` 保存可以删掉，或者保留作为兜底，但核心应该靠这个 effect。

## 四、恢复文章时还有一个低效但不一定导致 bug 的点

你现在在循环里每个 id 都调用一次：

```ts
const articles = await window.electronAPI.getArticles({})
const a = articles.find((aa: Article) => aa.id === id)
```

这会重复查很多次，而且 `getArticles({})` 默认只返回 20 篇。如果保存队列里有超过 20 篇，后面的文章可能根本查不到。`ipc-handlers.ts` 里 `articles:get` 默认 `limit = 20`，这会影响恢复完整队列。

至少改成只查一次，并提高 limit：

```ts
const articles = await window.electronAPI.getArticles({ limit: 500 })
const articleMap = new Map(articles.map((a: Article) => [a.id, a]))

for (const id of saved.ids) {
  if (blockedSet.has(id)) continue
  const a = articleMap.get(id)
  if (!a) continue

  savedItems.push({
    articleId: a.id,
    title: a.title || '',
    titleCn: a.title_cn || '',
    abstract: a.abstract || '',
    abstractCn: a.abstract_cn || '',
    doi: a.extra_doi || '',
    tldr: '',
    innovation: [],
    journal: a.journal || '',
    year: a.year || 0,
    score: 0,
  })
}
```
